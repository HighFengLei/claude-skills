# karpathy-guidelines

**来源**: multica-ai/andrej-karpathy-skills  
**许可**: 未明确标注（社区贡献）

## 作用

提供编码行为准则，减少 LLM 常见的编码错误。来自 Andrej Karpathy 的经验总结喵。

## 适用场景

- 任何需要 Claude 编写或修改代码的任务喵
- 希望代码质量更高、变更更精准时喵
- 团队协作中希望减少不必要的代码改动时喵

## 核心要点

1. **先思考再编码**：明确假设，有疑问就问，有更简单方案就说喵
2. **保持简单**：不多写一行不需要的代码，不提前抽象喵
3. **精准修改**：只改动必要部分，不重构不相关代码喵
4. **目标驱动**：把任务转化为可验证的目标，循环直到通过喵
